# FFMall git提交日志规范
###1.	提交日志基本格式
```text
<type>(<scope>):<subject>
<空行>
<body>
<空行>
<footer>
```
首行必填，其他选填。 `<subject> <body> <footer>`默认使用中文。
###2.	header部分
####2.1 type
|  type   | describe  |
|  ----  | ----  |
| feat     | 用户的新功能，而不是构建脚本的新功能      |
| fix      | 对用户错误的修复，而不是构建脚本的修复    |
| docs     | 对文档的修改                         |
| styles   | 代码格式化改动                        |
| refactor | 重构代码                            |
| test     | 新增和修改测试用例                    |
| other    | 其他                               |

####2.2 scope
可以为空，指明修改内容的范围。
####2.3 subject
概况一次提交行为囊括的内容
•使用现在时
•不要增加主语
•句尾无需标点，英语句首无需大写。
###3.	body部分
日志的内容主体body，用来描述详细的提交内容。用于补充subject内容，可不写。

###4.	footer部分
日志内容的页脚信息，用来描述一些补充信息，可不写。

###5.	实例
```text
docs:新增通用规范 git提交日志规范

用于规定git提交日志规范，便于提交日志工具分析和数据统计。
```
